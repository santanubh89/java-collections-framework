ConcurrentHashMap in Java is a thread-safe variant of HashMap that allows concurrent access and modifications by multiple threads without the need for external synchronization. It is part of the java.util.concurrent package and is designed to provide high concurrency and performance in multi-threaded environments.

Upto Java 7, ConcurrentHashMap used a technique called "lock striping" or "Segment based lock" where the map was divided into segments, each protected by its own lock. This allowed multiple threads to access different segments of the map simultaneously, improving concurrency. 16 segments were used by default, but this could be configured during the creation of the ConcurrentHashMap. These segments acted as independent hash maps, and each segment could be locked separately, allowing multiple threads to read and write to different segments concurrently. Only the segment being modified would be locked, while other segments remained accessible to other threads. Read operations were generally lock-free, allowing multiple threads to read from the map simultaneously without blocking each other, unless a write operation was in progress on the same segment.

Since Java 8, ConcurrentHashMap has been redesigned to use a more fine-grained locking mechanism based on a combination of lock-free techniques and synchronized blocks. The map is divided into bins (or buckets), and each bin can be accessed independently. The implementation uses a combination of volatile variables, compare-and-swap (CAS) operations, and synchronized blocks to ensure thread safety while minimizing contention. This allows for higher concurrency and better performance, especially in scenarios with a high number of read operations. In this approach, there is no fixed number of segments; instead, the map dynamically manages its internal structure to optimize for concurrent access.
Compare and Swap - A low-level atomic operation used to achieve synchronization without locking. It compares the current value of a variable with an expected value and, if they match, updates the variable to a new value. This operation is performed atomically, meaning it is indivisible and cannot be interrupted by other threads. CAS is commonly used in concurrent programming to implement lock-free data structures and algorithms, allowing multiple threads to safely update shared variables without the need for traditional locks, which can lead to contention and reduced performance.
